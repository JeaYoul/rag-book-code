# 1장 — 5분 만에 돌아가는 가장 단순한 RAG

RAG의 전부는 두 단계입니다. **"먼저 찾고(검색), 그다음 답한다(생성)."**
이 폴더의 코드는 그 두 단계를 30줄로 보여줍니다. 벡터 데이터베이스도, 프레임워크도 없습니다.

## 내 노트북으로 되나요?

**됩니다.** 그래픽카드(GPU)가 없어도 돌아갑니다. 최근 5~6년 안에 산 노트북이면 대부분 충분합니다.

- 운영체제: 윈도우 10(22H2)·11 / macOS 11+ / 리눅스
- 메모리: 8GB 이상
- 저장공간: 여유 10GB 이상

## 준비

**1) Ollama 설치** — [ollama.com](https://ollama.com)에서 내려받아 설치합니다. (리눅스는 `curl -fsSL https://ollama.com/install.sh | sh`)

**2) 모델 두 개 내려받기** — 합쳐서 약 3GB, 처음 한 번만 받으면 됩니다.

```bash
ollama pull bge-m3          # 임베딩용, 약 1.2GB
ollama pull qwen2.5:3b      # 생성용,   약 1.9GB
```

**3) 파이썬 패키지**

```bash
pip install ollama numpy
```

## 실행

```bash
python simple_rag.py
```

## 무슨 일이 일어나는가

질문을 벡터로 바꾸고 → 문서 다섯 개 중 의미가 가장 가까운 두 개를 찾고 → 그 자료를 근거로 답을 씁니다.

출력에서 두 가지를 눈여겨보세요.

**하나.** 답이 **자료에서** 나왔습니다. LLM이 지어낸 게 아닙니다. 문서를 바꾸면 답도 바뀝니다. 이것이 RAG가 환각을 줄이는 원리입니다.

**둘.** 문서 목록에 일부러 심어둔 엉뚱한 문장("담양은 전라남도에 있는 지역이다")은 **딸려오지 않습니다.** 질문과 의미가 멀어 검색 단계에서 걸러졌습니다. 오픈북의 수백 페이지 중 딱 필요한 페이지만 펼치는 것 — 이게 검색이 하는 일입니다.
