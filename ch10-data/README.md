## 10장의 핵심 — 한꺼번에 붓지 마라

> 빈 병에 물을 확 부으면 밖으로 넘치고 안엔 조금 들어간다. 천천히, 나눠서 부어야 다 담긴다.

그래서 **한 분야씩** 이 사이클을 돈다:

```
분야 선택 → ① 검색+메타데이터 → 목록 → ② 선별 → ③ 다운로드(nxml 우선/pdf) → ④ 검증 → DB
   └──────────────────────  다음 분야로 반복  ──────────────────────┘
```

(설포라판 → 글루코라파닌 → 브로콜리새싹 → 낙산균 → 단쇄지방산 … 한 분야씩)

## 파일 목록 (수집 4단계)

| 파일 | 단계 | 무엇 |
|---|---|---|
| `01_search_pubmed.py` | ① 검색+목록 | PubMed 한 분야 검색 → 메타데이터 함께 받아 목록 저장 |
| `02_select.py`        | ② 선별 | 목록을 보고 받을 대상만 필터 (연도·중복·결손 제거) |
| `03_download.py`      | ③ 다운로드 | 확정 목록으로 PMC 본문 받기 (**nxml 우선, 없으면 pdf**) |
| `04_verify.py`        | ④ 검증 | NCBI에서 서지정보 재수집·대조, 제목·저자·DOI 점수제 |
| `config.example.env`  | 공통 | 이메일·API 키·경로 등 설정 (복사해서 `.env`로) |

> 📌 파싱·DB 적재 코드는 여기 없습니다. 그것은 **11장(DB 설계)**, **12장(파싱 파이프라인)** 몫입니다.
>    10장은 "받아서 검증하는 것"까지입니다.

## 준비물

```bash
pip install biopython requests python-dotenv
```

- **Biopython** — NCBI Entrez(E-utilities) 접근용. PubMed 검색·메타데이터·다운로드에 쓴다.
- NCBI는 **이메일 등록**을 요구하고, **API 키**가 있으면 속도 제한이 완화된다.
  (무료 발급: NCBI 계정 → Settings → API Key Management)

## 실행 순서

```bash
# 0. 설정 준비
cp config.example.env .env
#   .env 를 열어 <이메일>, <NCBI_API_KEY>, 저장 경로를 채운다

# 1. 한 분야 검색 → 목록 만들기
python 01_search_pubmed.py --query "<당신의_검색어>" --years 10 --out lists/<분야>.csv

# 2. 목록에서 받을 대상 선별
python 02_select.py --in lists/<분야>.csv --out lists/<분야>_selected.csv

# 3. 선별 목록 다운로드 (nxml 우선, 없으면 pdf)
python 03_download.py --in lists/<분야>_selected.csv --dir data/<분야>

# 4. 메타데이터 보강·무결성 검증
python 04_verify.py --in lists/<분야>_selected.csv --dir data/<분야> --out lists/<분야>_verified.csv
```
